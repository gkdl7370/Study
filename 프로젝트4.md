# K-water National Data Sharing & Platform Integration (KWNSPI)

## 1. 프로젝트 개요

- **프로젝트 영문명:** K-water National Data Sharing & Platform Integration  
- **코드:** **KWNSPI**
- **발주처:** 한국수자원공사(K-water)
- **과업명:** <span style="color:brown">**데이터 공유ㆍ개방ㆍ서비스 통합 및 국가 플랫폼 연계 용역**</span>
- **과업기간/금액:** 착수일로부터 360일(12개월)<span style="color:brown">**(2025.11.04.~2026.10.29.)**</span>, **325,000,000원(VAT 포함)**

## 2. 과업 목적

### 2.1 유사 시스템 통폐합을 통한 비용 절감
기능이 유사한 정보시스템과 신규 기관공유데이터 관리시스템을 통폐합하여 불필요한 비용을 절감한다.

| 구분 | 시스템 | 역할 | 비고 |
|---|---|---|---|
| 서비스 | 물정보포털 | 물관련 콘텐츠, 데이터 대국민 서비스 | '16.01 - MyWater |
| 서비스 | 원클릭데이터 | GIS 기반 데이터 대국민 서비스 | '24.07 - KDM |
| 개방 | 공공데이터 | 데이터 대국민 개방(API) | '14.07 |
| 공유 | 물정보유통관리 | 기관간 데이터 공유(WINS, NDMS) | '20.01 |

### 2.2 데이터 수집·공유·개방 일원화 및 국가 플랫폼 연계
데이터 수집ㆍ공유ㆍ개방을 일원화하고, 국가공유데이터 플랫폼과 통합ㆍ연계하여 범정부 데이터 활용을 활성화한다.

## 3. 추진 방향

1. 데이터 수집ㆍ공유ㆍ개방을 일원화하고 설치된 기관공유데이터 관리시스템과 연계하여 국가공유데이터 관리시스템과 정보 및 데이터를 공유한다.  
2. 데이터 중복 및 오류를 방지하기 위해 **5개 유사정보 시스템**의 산재된 데이터 원천을 통합하여 데이터 모델을 설계하고 DB 구축 및 유사정보 시스템과 연계한다.  
   - 5개 유사정보 시스템: **물정보포털, 원클릭데이터, 공공데이터, 물정보유통관리, 기관공유데이터 시스템**
   - **물정보포털** : https://www.water.or.kr
   - **원클릭데이터** : https://www.water.or.kr/kdm/mainMap/main.do
   - **공공데이터 개방포털** : https://opendata.kwater.or.kr
3. 최신 클라우드 기술을 활용하여 **소스코드 통합 없이** 5개 유사정보 시스템을 통합하고 클라우드로 전환한다.  
4. 모바일 수문 서비스, AI 뉴스 추천, API 관리 기능 도입, 검색엔진 최적화, 다국어 검색 노출, 모바일 최적화 등 **접속자 수 증가를 위한 기존 서비스 기능 및 웹 접근성**을 개선한다.

## 6. 기능 요구사항 상세

### [SFR-001] - 기관공유데이터 관리시스템 연계(공유/개방 데이터의 수집ㆍ관리ㆍ제공)
- API Gateway 기반으로 체계적인 내ㆍ외부 API 관리
	- (공통) 모든 데이터 수집 및 제공 기능을 K-water에 도입될 APIGateway 또는 오픈소스(Tyk) 등을 기반으로 통합 관리해야 한다.
	- (수신) 기상청, 한국농어촌공사 등 외부기관 API를 10개 이상 수집해야 하며 신규 API 데이터가 추가될 수 있도록 변환 프로세스를 통해 공유 DB 주 스키마에 정제 데이터가 적재되도록 구현해야 한다.
	- (수신) 여러 API에서 수집한 제원, 코드 등 데이터를 기관메타관리표준 형태로 표준화하여 공유 DB에 통합해야 한다.
	- (송신) 공유ㆍ개방 DB 데이터를 SFR-002 민감도에 근거하여 공유 데이터로 설정한 데이터만 전송할 수 있도록 CSV, JSON 등 파일로 저장이 가능해야 하며 로컬 경로 또는 별도의 파일 서버로 유연하게 지정할 수 있도록 구현해야 한다. 동시에 해당 데이터를 실시간으로 조회ㆍ추가ㆍ수정할 수 있는 API를 자동으로 생성해야 한다.
	- (메타) 코드만 있어도 관련된 모든 정보가 올바르게 연결되어 조회 가능해야 하며 여러 종류의 코드(국가코드, 기관코드, 시스템 코드 등)가 실제로 같은 대상을 가리키는 경우에는 서로 다른 코드라도 하나로 매핑하여, 어떤 코드로 조회하더라도 동일한 정보가 정확하게 조회될 수 있도록 구현해야 한다.
	- (관리) URL 설정, 입력/출력 파라미터 정의, CSV·XML·JSON 변환 지원을 포함한 API 관리 기능을 제공해야 한다.
	- (관리) SQL 기반 입출력 테이블 설정, Swagger UI를 통한 문서화, 인증·로깅·라우팅·부하 분산 등 핵심 기능을 일관되게 처리해야 한다.
- INR-002(시스템 인터페이스), INR-003(사용자 인터페이스)의 요구사항을 준수해야 한다.

### [SFR-002] - 통합 데이터 모델 설계 및 구축
- 원본 데이터 소스 관리 기능
	- 관리자가 MV/View의 원본 DB 서버 정보를 등록하고 관리할 수 있도록 원본 테이블별 소유자, 스키마, 테이블명을 체계적으로 관리해야 한다.
	- 원본 DB 연결 상태 및 접근 권한 정보를 실시간으로 확인할 수 있어야 한다.
- MV/View 구조 정보 관리 기능
	- 생성된 MV/View별 구조 정보(컬럼, 데이터 타입, 제약조건), MV refresh 방식(Complete/Fast) 및 주기 설정을 화면에서 관리할 수 있어야 한다.
	- 코드 테이블 매핑 정보 및 표준화 규칙을 등록하고 수정할 수 있어야 한다.
	- 원본 테이블 변경 시 관련 MV/View 목록을 제공해야 한다.
- 원천 데이터 통합 관리(원천 식별)
	- 5개 시스템의 산재된 데이터 원천을 식별하고 통합해야 한다.
	- 원본 DB 스키마, API 데이터 비교 및 중복 데이터를 식별해야 한다.
	- 데이터 민감도를 공개/기관 공유/내부로 분류해야 한다.
- 데이터 이원화 구조 설계(개방 DB)
	- 공유 DB에서 국민에게 공개 가능한 데이터만 개방 DB로 구축해야 한다.
	- 데이터 보호 및 중복저장 방지를 위해 읽기 전용으로 구성해야 한다.
	- Oracle View 기반으로 개방 DB를 구현해야 한다.
- 데이터 이원화 구조 설계(공유 DB)
	- 기관 간 데이터 공유를 목적으로 하는 데이터를 공유 DB에서 관리해야 한다.
	- 물 관련 내·외부 데이터를 통합 저장 및 관리할 수 있어야 한다.
	- Materialized View(MV) 기반으로 공유 DB를 구현해야 한다.
- 원천 데이터 통합 관리(마이그레이션)
	- 표준화된 컬럼명과 데이터 형식을 정의해야 한다.
	- Complete/Fast Refresh 방식 등 적절히 선택하여 MV 생성 전략을 수립해야 한다.
	- 원본 DB 테이블을 참조하는 MV 생성 및 실시간 동기화를 위한 MV로그를 구성해야 한다.
- INR-001(시스템 연계 및 인터페이스 관리), INR-003(사용자 인터페이스), DAR-002(데이터 구조 설계 및 검증), DAR-003(데이터 구조 설계 및 검증) 및 TER-003(데이터 통합 및 정합성 테스트), SER-003(데이터접근 권한 관리), SER-004(데이터 암호화 및 보안) 요구사항을 준수해야한다.

### [SFR-003] - 시스템 통합 및 클라우드 전환
- 컨테이너 기반 시스템 구축(기존 시스템 컨테이너화)
	- 물정보포털, 원클릭데이터, 공공데이터, 물정보유통, 기관공유데이터 시스템을 개별 도커 컨테이너로 변환해야 한다.
	- 각 시스템을 독립적인 환경으로 분리하여 장애 확산을 억제할 수 있어야 한다.
	- Dockerfile 작성 및 의존성 라이브러리 패키징을 수행해야 한다.
- 컨테이너 기반 시스템 구축(컨테이너 관리 체계 구축)
	- 다중 컨테이너 관리도구(Kubernetes)를 활용해 자동 배포를 구성해야 한다.
	- 시스템 상태를 실시간으로 모니터링하여 자동으로 재시작할 수 있어야 한다.
	- 트래픽을 지능적으로 분산하여 시스템 안정성을 향상시켜야 한다.
- API Gateway 구현을 통한 단일 도메인 통합
	- 5개 시스템을 하나의 인터넷 주소(도메인)로 통합해야 한다.
- CI/CD 파이프라인 구축 및 자동화된 테스트/배포/운영 체계 구현- Git CI/CD를 활용한 자동화된 빌드 및 테스트 환경을 구축해야 한다.
	- 소스 코드 관리 및 이슈 트래킹 기능을 제공해야 한다.
	- 자동화된 테스트를 통한 품질 향상 및 오류 감소 기능을 구현해야 한다.
	- 무중단 배포 기능을 구현하여 서비스 중단을 최소화해야 한다.
	- 롤백 기능을 제공하여 오류 발생 시 이전 상태로 복원할 수 있어야 한다.
	- Kubernetes 대시보드 등을 통한 자동화된 모니터링 및 실시간 이슈 알림 기능을 제공해야 한다.
- 기타 서비스 전환(ArcGIS server, 동영상 스트리밍 서비스 등)
	- 기존에 설치되어있는 ArcGIS server를 클라우드 서버로 이관 해야한다.
	- 구형 동영상 서비스를 도커 기반 오픈소스 솔루션(예: KalTura)으로 전환해야 한다.
- TER-002(클라우드 전환 및 컨테이너 테스트),TER-004(사용자 경험 및 기능 개선 서비스 테스트) TER-005(재해복구 및 가용성 테스트)를 준수해야 한다.

### [SFR-004] - 기존 서비스 기능 개선 및 고도화
- 모바일 수문 서비스 및 AI 뉴스 추천 기능 개발
	- 모바일 대시보드에서 수문, 수위 등 기본 요약 정보를 제공해야 한다.
	- 추가 정보가 필요한 경우 물정보포털 페이지로 이동할 수 있어야 한다.
	- 반응형 웹 디자인을 적용하여 다양한 화면 크기와 기기에 최적화되어야 한다.
	- React 기반 또는 최적의 기술을 활용하여 사용자 중심 UI/UX를 설계해야 한다.
	- 외국 물 관련 뉴스를 수집해야 한다.
	- 생성형 AI를 활용한 최적의 뉴스 구조화 방안을 수립하여 감독원의 허가를 득한 후 기능을 개발해야 한다.
	- 지역별·주제별 뉴스를 추천하고 관련 뉴스를 모바일 대시보드에 제공해야 한다.

# META_OWNER로 접속 후 뷰 생성
CREATE OR REPLACE VIEW META_OWNER.V_SCHEMA_COLUMNS AS
SELECT
    C.OWNER        AS OWNER,
    C.TABLE_NAME   AS TABLE_NAME,
    TC.COMMENTS    AS TABLE_COMMENT,
    C.COLUMN_NAME  AS COLUMN_NAME,
    CC.COMMENTS    AS COLUMN_COMMENT,
    CASE
        WHEN C.DATA_TYPE IN ('CHAR', 'NCHAR', 'VARCHAR2', 'NVARCHAR2') THEN
            C.DATA_TYPE
            || '('
            || CASE
                   WHEN C.CHAR_USED = 'C' THEN C.CHAR_COL_DECL_LENGTH
                   ELSE C.DATA_LENGTH
               END
            || ')'
        WHEN C.DATA_TYPE = 'NUMBER' THEN
            CASE
                WHEN C.DATA_PRECISION IS NULL
                 AND C.DATA_SCALE IS NULL THEN
                    'NUMBER'
                WHEN C.DATA_PRECISION IS NOT NULL
                 AND (C.DATA_SCALE IS NULL OR C.DATA_SCALE = 0) THEN
                    'NUMBER(' || C.DATA_PRECISION || ')'
                ELSE
                    'NUMBER(' || C.DATA_PRECISION || ',' || C.DATA_SCALE || ')'
            END
        WHEN C.DATA_TYPE = 'FLOAT' THEN
            CASE
                WHEN C.DATA_PRECISION IS NOT NULL THEN
                    'FLOAT(' || C.DATA_PRECISION || ')'
                ELSE
                    'FLOAT'
            END
        WHEN C.DATA_TYPE = 'RAW' THEN
            'RAW(' || C.DATA_LENGTH || ')'
        ELSE
            C.DATA_TYPE
    END AS COLUMN_TYPE,
    C.COLUMN_ID
FROM SYS.DBA_TAB_COLUMNS C
	JOIN SYS.DBA_TABLES T ON T.OWNER = C.OWNER AND T.TABLE_NAME = C.TABLE_NAME
	LEFT JOIN SYS.DBA_TAB_COMMENTS TC ON TC.OWNER = C.OWNER AND TC.TABLE_NAME = C.TABLE_NAME
	LEFT JOIN SYS.DBA_COL_COMMENTS CC ON CC.OWNER = C.OWNER AND CC.TABLE_NAME = C.TABLE_NAME AND CC.COLUMN_NAME = C.COLUMN_NAME
WHERE 1 = 1
	AND C.OWNER IN ('DIS','EXWGIOS','HDAPS','KDM','NBEAVER','NED','OPDSEL','RWIS','SDAON','SGIOS','SGIOS_F','STP','WBP','WFM','WID','WPT','WQE','WRI','WRO')
	AND C.TABLE_NAME NOT LIKE 'BIN$%'
ORDER BY
    C.OWNER,
    C.TABLE_NAME,
    C.COLUMN_ID;
